01 Раздел

Что такое UMRABOR

UMRABOR — платформа, через которую зиёратчилар Узбекистана бронируют умра-туры у лицензированных тур-операторов. Точка входа — Click Mini App (~10 млн пользователей) и публичный веб-сайт. Платформа берёт на себя витрину, бронирование, оплату и инфраструктуру; тур-операторы — собственно туры (отели, авиа, виза, ваучеры).

Проблема

  • Тур-операторы продают туры через Telegram-чаты и Instagram — нет единой витрины
  • Клиент не понимает что именно входит в пакет, какие даты доступны, сколько стоит размещение разных типов
  • Размещение в номерах (с махрам / без, по полу, по семьям) — сложные правила, операторы делают руками в Google Sheets
  • Оплата идёт переводами, нет атомарности «оплата ↔ бронь»
  • Партнёр-агент / B2B не видит остатки в реальном времени

Решение

  • Единый каталог умра-пакетов с прозрачными ценами в UZS
  • Алгоритмическое размещение в номерах — клиент выбирает только тип (DBL/TRPL/QUAD), система назначает номер по правилам махрам и пола
  • Atomic payment-or-nothing через Click — окно оплаты 15 минут, без оплаты заказа не существует
  • Партнёрский кабинет с мастером публикации, остатками, заявками, чатом с клиентом
  • Команда UMRABOR модерирует пакеты и партнёров, видит финансы платформы

Аудитория

СегментОписание
ЗиёратчиларУзбекистанцы 25–65 лет, планирующие умра. Заходят через Click, говорят на узбекском (кириллица/латиница) или русском.
Тур-операторыЛицензированные турфирмы. Хотят продавать через массовый канал без своей разработки.
B2B-агентыПерепродают туры. Видят витрину как black box: типы с пометкой «доступно/недоступно».

03 Раздел

Ключевые сущности

Все 4 интерфейса работают с одним и тем же набором сущностей. Изменение в одном — отражается во всех остальных.

Иерархия данных

Пакет (Package) └── Вылет (Departure) — конкретная дата └── Группа (Group) — до 4 групп на вылет, по 45–50 чел. └── Номер (Room) — пул QUAD / TRPL / DBL └── Слот (Slot) — конкретное место с занятостью

Карточки сущностей

Пакет

Тур, который продаёт партнёр. Состав: фото, описание, маршрут (Тошкент → Маккаи Мукаррама → Мадинаи Мунаввара), услуги, подарки, питание, трансфер. Один пакет — много вылетов.
name · type · route · services · gifts · meals · photos

Вылет

Конкретная дата с конкретной авиакомпанией, рейсом, общей ёмкостью, ценой по типам размещения, остатком инвентаря. Партнёр задаёт ёмкость по группам.
date · airline · capacity · prices · groups[]

Инвентарь

Пул номеров на вылет: QUAD ×N, TRPL ×N, DBL ×N. Состояние номера: EMPTY / MALE_ONLY / FEMALE_ONLY / FAMILY_LOCKED. Партнёр ведёт вручную, обновляет в начале дня.
type · capacity · occupied · gender_state · source

Зиёратчи

Паломник. Поля: ФИО, телефон, паспорт (фото), даты, пол, виза. Махрам — опциональная связь с другим зиёратчи в этой же заявке. Срок паспорта < 6 мес. до вылета — блок.
name · passport · gender · mahram_link · visa

Заявка (Буюртма)

Бронь. Создаётся клиентом в мини-аппе/вебе. Lifecycle: paid_pending_op → processing → completed (или cancelled_op / cancelled_user). Lock инвентаря на 15 мин для оплаты.
id · package · departure · pilgrims · room · status

Тўлов

Платёж через Click Mini App (Visa/Mastercard/Uzcard/Humo через Click). Atomic payment-or-nothing (D-34). Окно 15 минут (D-35). Idempotency через Click transaction_id.
click_tx · amount · status · created_at

Ваучер

PDF на каждого зиёратчи. Генерируется после подтверждения заявки оператором. Содержит пакет, дату вылета, тип размещения, ФИО, соседей по номеру (свои — открыто, чужие — XXXXX).
order_id · pilgrim · type · pdf_url

Партнёр

Тур-оператор. Имеет логотип, фото офиса, лицензию, описание, реквизиты. Команда: Админ + Операторы. Все пакеты и заявки привязаны к партнёру.
name · license · contacts · team[] · packages[]
Правило размещения: клиент выбирает тариф (DBL/TRPL/QUAD) и вариант (вместе/раздельно для махрам), а конкретный номер назначает алгоритм по водопадной логике: уплотнение → новый номер → следующая группа.
04 Раздел

End-to-end флоу — от публикации до ваучера

Сценарий запуска одного пакета и одной брони, проходящий через все 4 интерфейса и 3 действующих лица: партнёра, команду UMRABOR и клиента.

● Партнёр (Админ)
АРМ Партнёра · Пакетлар → +Янги пакет

Партнёр публикует пакет

Запускает 4-шаговый мастер публикации:

  • Шаг 1 — Маълумотлар: название, тип (Эконом/Стандарт/Премиум/Люкс), фото, маршрут (остановки + ночи + гостиница), услуги, подарки, питание, тип трансфера
  • Шаг 2 — Вылеты: создаёт даты с авиакомпанией, рейсом, ёмкостью групп (по умолчанию 45 чел./группа, 4 группы)
  • Шаг 3 — Инвентарь: задаёт стандартный набор номеров (5×QUAD + 5×TRPL + 5×DBL = 45 мест) и тарифы (цена за QUAD / TRPL / DBL per person)
  • Шаг 4 — Подтверждение: сводный экран всех данных, кнопка «Опубликовать»
● Команда UMRABOR
Платформа UMRABOR · Модерация → Пакеты

Платформа модерирует пакет

Пакет приходит в очередь модерации со статусом pending_moderation. Модератор открывает drawer пакета с 3 вкладками: Информация / Инвентарь / История.

  • Проверяет: фото качественные, маршрут логичный, цены не аномальные, услуги корректны, описание без ошибок
  • Одобряет → пакет переходит в active, появляется в каталоге
  • Отклоняет → выбирает причину из справочника, пишет комментарий партнёру
● Клиент
Click Mini App / Веб-сайт · Каталог

Клиент находит пакет

Заходит в UMRABOR через Click Mini App (или на веб) и видит каталог:

  • Карточка пакета: фото, имя, города + ночи + дата ближайшего вылета, ценовые тайлы (1/2/3/4 кишилик хона) — цены справа
  • Открывает страницу пакета: галерея, описание, меҳмонхона блоки (Маккаи Мукаррама + Мадинаи Мунаввара с золотым и зелёным pill), сафар йўналиши (таймлайн), хизматлар + совғалар, нархлар (4 строки), ҳожиларимиз тажрибаси
● Клиент
Click Mini App / Веб-сайт · Бронирование

Клиент бронирует — 4 шага

  • Шаг 1 — Зиёратчилар руйхати: добавляет себя и членов семьи (имя, фамилия, дата рождения, пол, паспорт). Опционально — «Загранпасортим йўқ» → ID-паспорт + помощь с оформлением. Маҳрам блок появляется со 2-го зиёратчи.
  • Шаг 2 — Кунни танлаш (вылет): выбирает дату из доступных вылетов. Каждый вылет — карточка с датой, авиакомпанией, ценами 1/2/3/4 кишилик.
  • Шаг 3 — Жойлашув: система предлагает варианты размещения с учётом состава семьи (для семьи с махрамом — варианты «вместе» / «раздельно»). Радио-кнопки с ценой за номер + умумий сумма.
  • Шаг 4 — Тўлов: Жами тўлов (золотой) + выбор способа оплаты (Click / Payme / Paynet / Uzcard в вебе; в мини-аппе только Click) + agreement checkbox. Кнопка «Тўлаш».
● Клиент
Click Mini App · WebView оплаты

Оплата через Click — 15 минут окно

  • Открывается Click WebView с visual countdown timer (D-36). Push за 2 минуты, красный за 30 секунд
  • Инвентарь залочен на 15 минут (D-35). Без оплаты — заказа не существует (D-34)
  • Внутри окна разрешены неограниченные попытки оплаты (D-37)
  • При закрытии WebView — 60 сек grace period (D-38)
  • После успеха — статус paid_pending_op, экран успеха «Ассалому алайкум! Сафарингиз қабул қилинди»
● Партнёр (Оператор)
АРМ Партнёра · Заявки → drawer заявки

Оператор подтверждает заявку

Заявка падает в Заявки со статусом paid_pending_op. SLA подтверждения 30 мин — 24 часа.

  • Открывает карточку заявки: пакет, вылет, зиёратчилар, тариф, оплата, чат с клиентом
  • Проверяет паспорта (срок > 6 мес. от даты вылета), визу, корректность данных
  • Тасдиқлаш → статус processing, инвентарь списан окончательно
  • Отказ → причина из справочника → автовозврат через Click в течение 24 ч
● Партнёр (Оператор)
АРМ Партнёра · Заявка → Документы

Партнёр генерирует и прикрепляет ваучеры

После подтверждения партнёр готовит документы:

  • Подаёт на саудийскую визу, получает
  • Генерирует PDF-ваучер на каждого зиёратчи: тип пакета, дата вылета, тип размещения, ФИО, соседи (свои открыто, чужие XXXXX), блок Махрам если есть
  • Прикрепляет ваучеры к заявке в кабинете
  • Клиент получает уведомление
● Клиент
Click Mini App / Веб · Менинг буюртмаларим

Клиент скачивает ваучеры

  • В разделе «Менинг буюртмаларим» — детальная карточка заявки
  • 3 блока: Буюртма ҳолати (таймлайн этапов с галочками), Ваучерлар (список PDF с кнопкой скачивания), Буюртма тавфсилоти (полные данные)
  • Если что-то непонятно — открывает Қўллаб-қувватлаш (чат с оператором UMRABOR)
● Команда UMRABOR
Платформа UMRABOR · Платежи

UMRABOR получает комиссию

  • В разделе Платежи отображается транзакция: партнёр, пакет, сумма, комиссия UMRABOR, Click-комиссия, чистая выплата партнёру
  • KPI: Жами тушум, Жами комиссия, Бугунги, Бекор қилинган
  • Выплата партнёру по графику расчётов
05 Раздел

Роли — кто что видит и может

В системе 5 ролей. Каждая привязана к своему интерфейсу со своими правами.

Клиент (Зиёратчи) ФИЗ

Узбекистанец, заходит через Click

Бронирует туры для себя и семьи. Авторизация — Click аккаунт (телефон), без отдельной регистрации.

Может
  • Просматривать каталог, страницы пакетов, галерею
  • Создавать заявку с указанием зиёратчилар (включая других людей)
  • Оплачивать через Click
  • Видеть статусы своих буюртма, скачивать ваучеры
  • Писать в Қўллаб-қувватлаш
  • Менять язык интерфейса
umrabor_client_v6.html · umrabor_web_v1.html

Админ партнёра ЮР

Владелец турфирмы

Полный доступ к кабинету партнёра. Управляет всем что касается его турфирмы.

Может
  • Всё что может Оператор
  • Редактировать профиль партнёра (логотип, фото, лицензия, реквизиты)
  • Создавать и удалять пользователей в команде (Админ / Оператор)
  • Управлять справочниками (отели, авиа, услуги, подарки, тип пакета, инвентарь)
  • Настройки уведомлений (триггеры + каналы)
  • Видеть полную историю действий команды
umrabor_admin_v5.html

Оператор партнёра ЮР

Сотрудник турфирмы

Операционная работа: пакеты, заявки, зиёратчилар. Без управления командой и настройками.

Может
  • Публиковать и редактировать пакеты (4-шаговый мастер)
  • Управлять вылетами, ценами, инвентарём
  • Обрабатывать заявки: подтверждать, отказывать, прикреплять ваучеры
  • Видеть и редактировать карточки зиёратчилар
  • Просматривать дашборд (продажи, конверсии, остатки)
  • Общаться с клиентами в чате
umrabor_admin_v5.html (без Настроек и Команды)

Команда UMRABOR UMR

Сотрудник UMRABOR (модератор / админ)

Управляет всей платформой: модерирует контент партнёров, видит деньги, ведёт пользователей команды.

Может
  • Модерировать пакеты, гостиницы, новых партнёров
  • Видеть все заявки всех партнёров
  • Видеть финансы платформы: транзакции, комиссии UMRABOR
  • Управлять пользователями команды (роли: Админ UMRABOR / Модератор)
  • Просматривать историю действий команды UMRABOR (только своя команда)
  • Управлять дашбордом платформы (топ партнёров, топ пакетов, платежи)
umrabor_platform_v1.html

Аноним (посетитель веб-сайта) ANON

Незалогиненный пользователь веба

Заходит на публичный сайт UMRABOR. Может смотреть, не может покупать.

Может
  • Просматривать главную страницу с hero, видео, галереей, отзывами
  • Просматривать каталог и страницы пакетов
  • Видеть лендинг для тур-операторов и оставлять заявку на партнёрство
  • Нажать «Сотиб олиш» → откроется модалка Кириш (вход по телефону + SMS) → станет Клиентом
